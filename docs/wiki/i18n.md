# i18n (Internationalization) | 国際化

## Overview
The i18n module provides bilingual support (English / Japanese) for activity type names, property names, and UI strings displayed in the visualizer panel.

## 概要
i18nモジュールは、ビジュアライザーパネルに表示されるアクティビティタイプ名、プロパティ名、UI文字列のバイリンガルサポート（英語/日本語）を提供します。

```
packages/shared/i18n/i18n.ts
```

## Architecture | アーキテクチャ

### Language State | 言語状態

The current language is stored as a module-level singleton:

```typescript
type Language = 'en' | 'ja';
let currentLanguage: Language = 'en'; // Default: English
```

The state is shared across the entire shared library within a single browser page context.

現在の言語はモジュールレベルのシングルトンとして保存されます。状態は単一ブラウザページコンテキスト内でsharedライブラリ全体で共有されます。

### Language Persistence | 言語設定の永続化

The GitHub extension persists the user's language preference in `chrome.storage.sync`:

```typescript
// Load on startup
const result = await chrome.storage.sync.get('language');
if (result.language) setLanguage(result.language as Language);

// Save on change
await chrome.storage.sync.set({ language: lang });
```

`chrome.storage.sync` syncs the preference across devices when the user is signed into Chrome.

GitHub拡張機能は言語設定を`chrome.storage.sync`に永続化します。`chrome.storage.sync`はユーザーがChromeにサインインしている場合、デバイス間で設定を同期します。

## Translation Functions | 翻訳関数

### `setLanguage(lang: Language): void`
Sets the current language. Call this before rendering.

レンダリング前に呼び出して現在の言語を設定します。

### `getLanguage(): Language`
Returns the current language.

現在の言語を返します。

### `translateActivityType(type: string): string`
Translates an activity type name.
- English: returns the XAML value as-is (e.g., `Sequence`, `Assign`)
- Japanese: looks up `activityTypeMap`, falls back to English if not found

アクティビティタイプ名を翻訳します。
- 英語: XAMLの値をそのまま返す（例：`Sequence`、`Assign`）
- 日本語: `activityTypeMap`を参照し、なければ英語にフォールバック

### `translatePropertyName(name: string): string`
Translates a property name.
- English: returns the name as-is
- Japanese: looks up `propertyNameMap`, falls back to English if not found

プロパティ名を翻訳します。
- 英語: 名前をそのまま返す
- 日本語: `propertyNameMap`を参照し、なければ英語にフォールバック

### `t(key: string): string`
Translates a UI string key.
- Looks up `uiStringsMap[key][currentLanguage]`
- Falls back to the key itself if not found

UI文字列キーを翻訳します。
- `uiStringsMap[key][currentLanguage]`を参照
- 見つからない場合はキーをそのまま返す

## Translation Maps | 翻訳マップ

### Activity Type Map | アクティビティタイプマップ (`activityTypeMap`)

Japanese translations for activity types. English uses the XAML element name directly.

アクティビティタイプの日本語訳。英語はXAML要素名を直接使用。

| English (XAML) | Japanese |
|----------------|----------|
| `Sequence` | シーケンス |
| `Assign` | 代入 |
| `If` | 条件分岐 |
| `ForEach` | 繰り返し（ForEach） |
| `NClick` | クリック（モダン） |
| ... | ... |

### Property Name Map | プロパティ名マップ (`propertyNameMap`)

Japanese translations for property names.

プロパティ名の日本語訳。

| English | Japanese |
|---------|----------|
| `To` | 代入先 |
| `Value` | 値 |
| `Condition` | 条件 |
| `Target` | ターゲット |
| ... | ... |

### UI Strings Map | UI文字列マップ (`uiStringsMap`)

Both English and Japanese strings for UI labels.

UIラベルの英語・日本語両方の文字列。

| Key | English | Japanese |

| `Removed` | Removed | 削除 |
| `Modified` | Modified | 変更 |
| `Properties` | Properties | プロパティ |
| `Linked` | Linked | リンク済み |
| ... | ... | ... |

## Adding Translations | 翻訳の追加方法

### Adding a new activity type | 新しいアクティビティタイプの追加

```typescript
// In activityTypeMap:
'NewActivity': '新アクティビティ',
```

### Adding a new property name | 新しいプロパティ名の追加

```typescript
// In propertyNameMap:
'NewProperty': '新プロパティ',
```

### Adding a new UI string | 新しいUI文字列の追加

```typescript
// In uiStringsMap:
'My Label': { en: 'My Label', ja: 'マイラベル' },
```

Then use it in code with: `t('My Label')`

コードでは`t('My Label')`で使用します。

## Language Toggle | 言語切り替え

When the user changes the language in the panel header, the extension:
1. Saves the new language to `chrome.storage.sync`
2. Calls `setLanguage(lang)`
3. Re-renders the current panel with the new language

ユーザーがパネルヘッダーで言語を変更すると、拡張機能は：
1. 新しい言語を`chrome.storage.sync`に保存
2. `setLanguage(lang)`を呼び出す
3. 新しい言語で現在のパネルを再レンダリング
